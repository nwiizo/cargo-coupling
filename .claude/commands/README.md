# Custom Commands for cargo-coupling

このディレクトリには、cargo-couplingプロジェクトの開発・分析を支援するカスタムコマンドが含まれています。

## コマンド一覧

### 分析系

| コマンド | 説明 | 所要時間 |
|---------|------|----------|
| `/analyze` | カップリング分析を実行し、結果を解釈 | 2-5分 |
| `/check-balance` | バランススコアを素早く確認 | 1分 |

### レビュー系

| コマンド | 説明 | 所要時間 |
|---------|------|----------|
| `/full-review` | 総合アーキテクチャレビュー（推奨） | 10-15分 |

### リファクタリング支援

| コマンド | 説明 | 所要時間 |
|---------|------|----------|
| `/refactor` | 具体的なリファクタリング提案 | 5-10分 |
| `/explain-issue` | 問題タイプの詳細解説 | 2-3分 |

---

## 推奨ワークフロー

### 日常的な確認

```
/check-balance ./src
```

バランススコアを素早く確認し、問題がないかチェック。

### 詳細な分析が必要な場合

```
/analyze ./src
```

検出された問題の詳細と改善提案を取得。

### 総合的なレビュー（リリース前など）

```
/full-review ./src
```

3人の専門家による多角的なレビューを実行。

### リファクタリング実施時

```
/refactor ./src global-complexity
```

特定の問題タイプに対する具体的なリファクタリング手順を取得。

### 問題の理解を深めたい場合

```
/explain-issue cascading-change-risk
```

問題タイプの詳細解説と解決アプローチを確認。

## カスタマイズ

### 閾値の変更

分析時に閾値を変更できます：

```
/analyze ./src --max-deps 20 --max-dependents 25
```

### 特定の問題のみ分析

```
/refactor ./src high-efferent
```

---

## 注意事項

- コマンドはcargo-couplingのビルドが完了している前提で動作します
- Git履歴分析には十分なコミット履歴が必要です
- 大規模プロジェクトでは分析に時間がかかる場合があります
